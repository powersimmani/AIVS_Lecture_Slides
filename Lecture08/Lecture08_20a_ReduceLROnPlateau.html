ReduceLROnPlateau
Adaptive learning rate reduction when metric stops improving
How It Works
Core Idea
Monitor a metric (e.g., validation loss).
If no improvement for patience epochs,
reduce learning rate by factor.
ParameterDefaultDescription
mode'min''min' for loss, 'max' for accuracy
factor0.1LR multiplier when reducing
patience10Epochs to wait before reducing
threshold1e-4Min change to qualify as improvement
min_lr0Lower bound on learning rate
cooldown0Epochs to wait after reduction
Implementation
# PyTorch
from torch.optim.lr_scheduler import ReduceLROnPlateau

optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
scheduler = ReduceLROnPlateau(
  optimizer,
  mode='min', # minimize val_loss
  factor=0.5, # halve LR
  patience=5, # wait 5 epochs
  min_lr=1e-6, # don't go below
  verbose=True
)

# Training loop
for epoch in range(epochs):
  train_one_epoch()
  val_loss = validate()
  scheduler.step(val_loss) # Pass metric!
# Keras/TensorFlow
from tensorflow.keras.callbacks import ReduceLROnPlateau

callback = ReduceLROnPlateau(
  monitor='val_loss',
  factor=0.5,
  patience=5,
  min_lr=1e-6
)
model.fit(X, y, callbacks=[callback])
Best Practices
- Monitor validation metric, not training
- patience = 5-10 epochs typically
- factor = 0.5 or 0.1 (halve or 10x reduce)
- Combine with early stopping
