KV Cache: Key-Value Caching for LLM Inference
Essential optimization for autoregressive generation
The Problem
Autoregressive Generation:
Each new token requires attention over ALL previous tokens. Without caching, we recompute K and V for every token!
Without KV Cache (Naive):

Token 1: Compute K₁, V₁
Token 2: Compute K₁, V₁, K₂, V₂ ← redundant!
Token 3: Compute K₁, V₁, K₂, V₂, K₃, V₃ ← more redundant!
...
Token n: Recompute everything ← O(n²) total
Complexity Without Cache
Per token: O(n) attention computation
Total for n tokens: O(n²) computations
The Solution: KV Cache
Key Insight: K and V for previous tokens don't change! Cache them and only compute for the new token.
With KV Cache:

Token 1: Compute K₁, V₁ → Cache
Token 2: Load K₁, V₁, Compute K₂, V₂ → Cache
Token 3: Load K₁, V₁, K₂, V₂, Compute K₃, V₃
...
Token n: O(1) new computation per token
KV Cache Memory
Size = 2 × layers × batch × seq_len × d_model × dtype
Example (7B, 2048 tokens, FP16):
= 2 × 32 × 1 × 2048 × 4096 × 2 ≈ 1GB
Advanced Optimizations
• PagedAttention (vLLM): Efficient memory management
• MQA/GQA: Share K,V across heads
• Prefix Caching: Reuse for shared prefixes
