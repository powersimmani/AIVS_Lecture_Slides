Handling Class Imbalance
Strategies for skewed class distributions in classification
⚠️ Problem: Fraud detection has 99% normal, 1% fraud. A model predicting "always normal" gets 99% accuracy! Accuracy is misleading for imbalanced data.
1
Class Weights
Penalize misclassifying minority class more
Built into most sklearn classifiers
No data duplication needed
# Automatic balancing
LogisticRegression(
  class_weight='balanced'
)

# Manual weights
LogisticRegression(
  class_weight={0: 1, 1: 99}
)
2
Resampling
Oversampling: Duplicate minority samples
SMOTE: Generate synthetic minority samples
Undersampling: Remove majority samples
# pip install imbalanced-learn
from imblearn.over_sampling import SMOTE

smote = SMOTE(random_state=42)
X_res, y_res = smote.fit_resample(X, y)
3
Threshold Tuning
Default threshold 0.5 may not be optimal
Lower threshold → More positive predictions
Use PR curve to find best threshold
# Get probabilities
probs = model.predict_proba(X)[:, 1]

# Custom threshold
threshold = 0.3
preds = (probs >= threshold).astype(int)
