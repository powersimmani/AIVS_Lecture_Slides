Softmax Numerical Stability
The Log-Sum-Exp Trick
The Overflow Problem
softmax(z)i = exp(zi) / sumj exp(zj)
What Goes Wrong?
z = [1000, 1001, 1002]

exp(1000) = 1.97 × 10^434
exp(1002) = 7.31 × 10^435

Both exceed float64 max (~1.8 × 10^308)
Result: inf / inf = NaN
Underflow Too!
z = [-1000, -999, -998]

exp(-1000) ≈ 0 (underflow)

Result: 0 / 0 = NaN
# Naive implementation (UNSTABLE)
def softmax_naive(z):
  return np.exp(z) / np.sum(np.exp(z))

# Test
softmax_naive([1000, 1001, 1002])
# Output: [nan, nan, nan]
The Solution: Subtract Max
softmax(z) = softmax(z - max(z))
Mathematically equivalent, numerically stable!
Why It Works (Proof)
softmax(z - c)i = exp(zi - c) / sum exp(zj - c)
  = exp(zi)*exp(-c) / sum exp(zj)*exp(-c)
  = exp(zi) / sum exp(zj)
  = softmax(z)i

The exp(-c) cancels out! Works for any constant c.
# Stable implementation
def softmax_stable(z):
  z_shifted = z - np.max(z)
  exp_z = np.exp(z_shifted)
  return exp_z / np.sum(exp_z)

# Test
softmax_stable([1000, 1001, 1002])
# Output: [0.09, 0.24, 0.67] ✓
Good News
PyTorch, TensorFlow, JAX all handle this automatically!
Use F.softmax() or tf.nn.softmax() - they're stable.
Only worry if implementing from scratch.
