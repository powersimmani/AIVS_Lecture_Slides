Time Series Cross-Validation
Respecting temporal order in model evaluation
The Problem: Data Leakage
Why Standard K-Fold Fails
Standard CV randomly shuffles data.
For time series: future data leaks into training!

Train on 2023, test on 2022 -> unrealistic evaluation
Standard K-Fold (WRONG for time series)
Consequences
- Overly optimistic performance estimates
- Model fails in production (real future data)
- Autocorrelation causes train/test similarity
Time Series CV Methods
Expanding Window (TimeSeriesSplit)
Methods
Expanding: Train grows, test slides forward
Sliding: Fixed window moves forward
Blocked: Gap between train and test
# sklearn TimeSeriesSplit
from sklearn.model_selection import TimeSeriesSplit

tscv = TimeSeriesSplit(n_splits=5)
for train_idx, test_idx in tscv.split(X):
