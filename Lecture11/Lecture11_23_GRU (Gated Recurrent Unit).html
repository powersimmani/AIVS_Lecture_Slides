LSTM vs GRU: Key Differences
LSTM
  • üî¢ 3 gates: Forget, Input, Output
  • üíæ 2 states: Cell state (C) + Hidden state (h)
  • ‚öôÔ∏è More parameters to learn
  • üéØ Better for complex, long sequences
  • ‚è±Ô∏è Slower training time
  • üîß More hyperparameters to tune
GRU
  • üî¢ 2 gates: Update, Reset
  • üíæ 1 state: Hidden state (h) only
  • ‚öôÔ∏è Fewer parameters (~25-30% less)
  • üéØ Better for smaller datasets
  • ‚è±Ô∏è Faster training time
  • üîß Simpler to implement and tune

GRU Calculation (3D Vector Example)
Given Input:
x_t = [0.5, -0.3, 0.8] (current input)
h_{t-1} = [0.2, 0.4, -0.1] (previous hidden state)
Note: GRU has NO separate cell state - only hidden state!

r_t = œÉ(W_r ¬∑ [h_{t-1}, x_t] + b_r)
Controls how much past info to use for candidate
Result: r_t = [0.58, 0.73, 0.45]
Step 2: Update Gate (z_t)
z_t = œÉ(W_z ¬∑ [h_{t-1}, x_t] + b_z)
Combines forget & input: decides mix of old vs new
Result: z_t = [0.62, 0.48, 0.71]

hÃÉ_t = tanh(W_h ¬∑ [r_t ‚äô h_{t-1}, x_t] + b_h)

Step 4: Final Hidden State (h_t)

Key Insight: GRU achieves similar results with fewer operations: 3 steps vs LSTM's 4 steps, and 1 state vs LSTM's 2 states
