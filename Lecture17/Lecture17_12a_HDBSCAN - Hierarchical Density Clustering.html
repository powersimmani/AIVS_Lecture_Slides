HDBSCAN: Hierarchical Density-Based Clustering
DBSCAN's smarter sibling - no epsilon needed (Campello et al., 2013)
Why HDBSCAN?
DBSCAN Limitations
• eps selection: Requires manual tuning
• Varying density: Single eps can't handle different cluster densities
• No hierarchy: Flat clustering only
HDBSCAN Solution
• No eps needed: Automatically finds optimal density threshold
• Varying density: Handles clusters with different densities
• Hierarchical: Produces cluster hierarchy tree
• Robust outliers: Better noise detection

Practical Usage
Key Parameters
min_cluster_size (5-100): Minimum points to form a cluster. Start with intuition about smallest meaningful cluster.
min_samples (default=min_cluster_size): Conservative outlier detection. Higher = more points marked as noise.
cluster_selection_epsilon: Optional, similar to DBSCAN's eps for fine-tuning.
cluster_selection_method: 'eom' (excess of mass) or 'leaf' (flat clusters).
# pip install hdbscan
import hdbscan

# Basic usage - just set min_cluster_size!
clusterer = hdbscan.HDBSCAN(
  min_cluster_size=15,
  min_samples=5,
  metric='euclidean'
)
labels = clusterer.fit_predict(X)

# -1 = noise points
# Get cluster persistence/stability
clusterer.cluster_persistence_

# Soft clustering (probability)
clusterer.probabilities_
Best Practice
• Often pair with UMAP: reduce dimensions first, then HDBSCAN
• Use cluster_persistence_ to assess cluster quality
• -1 labels are outliers - embrace them, don't force assignment
