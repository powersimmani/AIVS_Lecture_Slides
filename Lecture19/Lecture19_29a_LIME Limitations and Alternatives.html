LIME Limitations & Alternatives
Understanding when LIME fails and what to use instead
LIME Limitations
1. Instability
Different random samples → different explanations
Same instance, different seeds = different feature importance
2. Kernel Width Sensitivity
Results highly depend on σ (kernel width) choice
No principled way to select optimal σ
3. Feature Independence Assumption
Samples features independently (ignores correlations)
May generate unrealistic data points
4. Local Fidelity Not Guaranteed
Linear surrogate may not accurately represent local behavior
Explanation can differ from model's actual reasoning
# Instability demonstration
exp1 = explainer.explain_instance(x, model,
  num_samples=1000, random_state=42)
exp2 = explainer.explain_instance(x, model,
  num_samples=1000, random_state=123)

# exp1 and exp2 may rank features differently!
Alternatives
Anchors (Rule-based)
Advantage: More stable, provides sufficient conditions
Output: IF-THEN rules (e.g., "IF age > 50 AND income < 30k THEN deny")
Use when: Need stable, human-readable rules
SHAP (Game-theoretic)
Advantage: Theoretical guarantees (Shapley values)
Output: Additive feature contributions
Use when: Need consistent, fair attribution
Integrated Gradients (Gradient-based)
Advantage: Satisfies sensitivity & implementation invariance
Output: Attribution per input dimension
Use when: Deep learning models
