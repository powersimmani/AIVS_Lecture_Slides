LIME: Mathematical Foundation
Local Interpretable Model-agnostic Explanations (Ribeiro et al., 2016)
Mathematical Formulation
xi(x) = argming in G L(f, g, pix) + Omega(g)
xi(x): Explanation for instance x
f: Original black-box model
g: Interpretable model (linear)
G: Class of interpretable models
L: Locality-aware loss
pi_x: Proximity measure to x
Omega(g): Complexity penalty
Loss Function
L(f, g, pi_x) = sum_z pi_x(z) * (f(z) - g(z'))^2
Weighted squared error between f and g on perturbed samples
Proximity Kernel
pi_x(z) = exp(-D(x,z)^2 / sigma^2)
Samples closer to x have higher weight
sigma = kernel width (hyperparameter)
Interpretable Model
g(z') = w_0 + sum_i w_i * z'_i
z': Binary interpretable representation
w_i: Feature importance weights
Intuition & Process
LIME Algorithm Steps
1. Perturb: Generate samples around x
2. Predict: Get black-box predictions for samples
3. Weight: Assign weights by proximity to x
4. Fit: Train weighted linear model
5. Extract: Coefficients = feature importance
Text LIME Example
Original: "This movie is wonderful!"
z' = [1, 1, 1, 1] -> Positive (0.95)

Perturbed: "This movie is ___!"
z' = [1, 1, 1, 0] -> Positive (0.60)

Perturbed: "This ___ is wonderful!"
z' = [1, 0, 1, 1] -> Positive (0.88)

Result: "wonderful" most important
Known Limitations
• Instability: Different runs can give different explanations
• Kernel width: Sensitive to sigma parameter
• Sampling: Random perturbations may miss important regions
• Independence: Assumes features are independent
• Fidelity: Linear model may not capture local behavior
