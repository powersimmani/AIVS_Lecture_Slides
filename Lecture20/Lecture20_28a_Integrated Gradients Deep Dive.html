Integrated Gradients: Axiomatic Attribution
Principled feature attribution for deep networks (Sundararajan et al., 2017)
Mathematical Foundation
IGi(x) = (xi - x'i) × integral01 (dF/dxi)(x' + alpha*(x-x')) d_alpha
x: input | x': baseline | F: model output | alpha: interpolation parameter
Two Fundamental Axioms
1. Sensitivity: If feature differs from baseline AND affects output, attribution != 0
2. Implementation Invariance: Two functionally equivalent networks get same attributions
Completeness Property
sumi IGi(x) = F(x) - F(x')

The attributions sum to the difference between the prediction and the baseline prediction. This ensures all prediction change is accounted for!
Why Not Just Gradients?
Vanilla gradient problems:
• Saturation: gradient = 0 in saturated regions
• Noisy: high-frequency patterns
• No baseline: relative to what?

IG fixes all three!

# Using Captum library
from captum.attr import IntegratedGradients

# Initialize
ig = IntegratedGradients(model)

# Compute attributions
attributions = ig.attribute(
  inputs=input_tensor,
  baselines=baseline_tensor, # zeros
  target=class_idx,
  n_steps=50 # Riemann approx steps
)

# For images: sum over channels
attr_sum = attributions.sum(dim=1)
n_steps Parameter
More steps = better integral approximation
• 20-50: Quick approximation
• 100-300: Good for final results
• Convergence check: increase until stable
IG vs Gradient Methods
Vanilla Gradient: Noisy, misses saturated regions
Gradient × Input: Better, but no axioms
SmoothGrad: Averages noise, but expensive
IG: Axiomatically justified, clean results
