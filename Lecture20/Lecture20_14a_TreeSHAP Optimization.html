TreeSHAP: Fast SHAP for Tree Models
Efficient exact computation of Shapley values (Lundberg et al., 2020)
Algorithm & Complexity
Key Innovation
Exploits tree structure to compute exact SHAP values efficiently.

Idea: Track feature subsets through all tree paths using dynamic programming instead of exponential enumeration.
MethodComplexityTime (100 features)
Exact ShapleyO(2m)Years
KernelSHAPO(2m) approxMinutes-Hours
TreeSHAPO(TLD2)Seconds
Complexity Breakdown
T: Number of trees
L: Maximum leaves per tree
D: Maximum depth

Polynomial in model size, NOT exponential in features!
Supported Models
• XGBoost, LightGBM, CatBoost
• sklearn: RandomForest, GradientBoosting, ExtraTrees
• Any tree-based model with accessible structure
Practical Usage
feature_perturbation Options
"tree_path_dependent" (default):
Fast, uses tree paths. May include feature correlations in attributions. Best for: speed, general use.
"interventional":
Causal interpretation, requires background data. Removes correlation effects. Best for: causal questions.
# Basic TreeSHAP usage
import shap

# For XGBoost/LightGBM/CatBoost
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_test)

# Interventional (causal) interpretation
explainer = shap.TreeExplainer(
  model,
  data=X_background, # ~100-1000 samples
  feature_perturbation="interventional"
)

# Visualizations
shap.summary_plot(shap_values, X_test) # global
shap.waterfall_plot(shap_values[0]) # local
shap.dependence_plot("feature", shap_values, X)
Performance Tips
• Use TreeExplainer (not KernelExplainer) for trees
• For large datasets: sample background data
• Batch processing: compute many at once
• Memory: shap_values can be large arrays
