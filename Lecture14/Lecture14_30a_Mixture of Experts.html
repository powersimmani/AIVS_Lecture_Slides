Mixture of Experts (MoE)
Scaling parameters without scaling compute
Architecture
Core Idea
Replace single FFN with multiple "expert" FFNs.
Router/Gate selects top-k experts per token.

8 experts, top-2 routing -> 8x params, ~2x compute!
MoE Layer
G(x) = Softmax(TopK(x · W_g))
y = sum_i G(x)_i · Expert_i(x)

Only top-k experts are computed!

Training Challenges
Load Balancing: All tokens go to same expert?
-> Add auxiliary loss to balance expert usage

Capacity: Expert overflow/underflow
-> Token dropping, capacity factor tuning
Benefits
+ Massive scale without proportional compute
+ Experts can specialize (languages, domains)
+ Better efficiency at inference
- Higher memory (all experts loaded)
- Training complexity (load balancing)
